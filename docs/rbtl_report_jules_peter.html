Abstract
Around 43% of worlds population does not have access to safe sanitation. In my PhD I will research a new approach to treat faeces using filamentous fungi. The goal of the research is to find fungal species that grow on human faeces and help reduce concentrations of faecal indicator bacteria. To be able to compare the growth speed and antibacterial effect of different fungal species a data analysis is necessary. With this report the way to manage and analyse the data was created.

Exploring fungi for faecal treatment

Introduction

Globally, 43% of the population lacks access to safe sanitation (Population Using Safely Managed Sanitation Services (, n.d.), leading to the spread of diarrheal diseases and environmental contamination from untreated faecal sludge. While urine diversion dry toilets (UDDTs) offer a potential solution by separating nutrient-rich and safe urine for fertilizer, the treatment of faeces is still a challenge. Long storage times and potential pathogen reactivation in dried faeces remain can be a risk for spreading of disease(Kumwenda et al., 2017).

The introduction of filamentous fungi, to sanitize faeces is a novel approach yet to be studied. Fungi are powerful decomposers in nature, utilizing enzymes to break down complex organic matter (Berg & McClaugherty, 2020). Coprophilous fungi, adapted to nutrient-rich dung, have evolved mechanisms to thrive in environments with high microbial competition, they are able to sense bacteria and produce antibacterial compounds up on recognition [Kombrink et al. (2019)]. Interestingly, preliminary research indicates that even non-dung-adapted white-rot fungi can colonize human faeces and reduce odour.

This doctoral thesis aims to investigate the potential of fungal species to utilize faeces as a nutrient source, reduce the concentration of harmful bacteria, and alter the physical properties of the faecal mass. Understanding these processes could lead to the development of an effective and sustainable method for treating faecal waste in UDDTs, mitigating health risks and shortening the required storage time for safe disposal or agricultural use.

Within this lecture of Research beyond the lab (RBTL) the code for data cleaning and analysis will be developed and tested using a example data set with simulated fungal growth and bacteria data. In order to test the capability of fungi to grow on human faeces, a petri dish based assay will be developed to measure growth speed and log-change of E. coli and Enterococcus before and after fungal over growth.

Methods

The data structure will be created in the following way. Figure 1

knitr::include_graphics("images/data_structure.jpg")

For the different task of the experiment separate tables will be generated.

  • faeces_1_1: Contains the data of the faecal collection creating one row of data for each faecal collection campaign.
  • inoculum_1_1: Stores the data for the inoculum used in the experiments.
  • bacterial_analysis_1_1: Contains the data about bacteria concentration after fungal co-incubation.
  • growth_speed_analysis_1_1: In this table each row will refer to a area size measured by image analysis for each picture
  • experiments_1_1: This table will store the experimental set up linking together all the tables using shared IDs

Tables will be created using google_sheets, and stored on google drive. Data analysis and figures are created using R (Rstudio V.2025.05.0 Build 496). An R-script for Loading the relevant data (01-data_download) another one cleans the data and prepare it for the analysis (02-data_cleaning.R). These R scripts are stored in the folder fungal-faecal-treatment/R. Data files being saved in the folder data processed with their corresponding name for the analysis as CSV files. The Q-Q plot Figure 2 indicates that the growth speed of C. comatus follows normal distribution, for Faecal Isolate 1 there is a slight deviation from the line which indicates some deviation from a normal distribution. For P. ostreatus the data seems to be skewed in both ends which suggest that the data does not follow normal distribution. The Shapiro test Table 1 on the data showed that P. ostreatus does not follow normal distribution.

shapiro_mean_growth_speed_species <- mean_growth_speed_species |>
  group_by(species)|>
  shapiro_test(mean_growth_speed)

shapiro_mean_growth_speed_species|>
  gt()|>
  tab_caption("Shapiro-test for testing mean growth speed of fungal species for normal distibution")

The shapiro test verifies the normal distribution for C. comatus and faecal isolate 1 while it shows violation of normal distribution for P. ostreatus v. Floridae. Figure 3 shows the median of the mean growth speed and distribution of the different fungal species in a Box-plot. Also here we can see that the median of P. ostreatus is shifted.

Since the mean growth speed is not normal distributed it would be recommended to use a test that allows not normal distributed data like Kruskal-Wallis test. This script should be able to handle both cases first an analysis of variance (anova) is done on the data set. To test if the mean growth of the different fungi are different from each other. The anova Table 2 with a Tukey post-hoc analysis analysis Table 3 shows that there is no significant difference in the mean growth speed between the different species for any of the pairs.

To complete the test setting the data will also be analysed using statistical tests that do not need normal distributed data. In this case a Kruskal-Wallis test Table 4 and a Dunn post-hoc analysis Table 5 are done

# Make sure 'species' is a factor
growth_speed_analysis <- growth_speed_analysis |>
  mutate(species = as.factor(species))

# Perform the Kruskal-Wallis test using rstatix
kruskal_result_growth_speed <- growth_speed_analysis |>
  kruskal_test(mean_growth_speed ~ species)


 kruskal_growth_speed_table <- kruskal_result_growth_speed|>
  select(-.y.) %>% # Remove the '.y.' column for cleaner output
  gt()
  
kruskal_growth_speed_table  
dunn_result <- growth_speed_analysis |>
  dunn_test(mean_growth_speed ~ species, p.adjust.method = "bonferroni")|>
  gt()

# Print the Dunn's test results
dunn_result

Conclusions

The produced data structure and scripts can be used on the test data set, to visualize mean growth speed differences between species and log-change of bacterial concentrations before and after fungal growth. The statistical tests ANOVA and Tukey can be used to test normal distributed data while Kruskal -Wallis and Dunn test can be used for the not normal distributed data. To test the data on the real created data will be a future challenge and adaptions have to be implemented depending on how the experiment setup will change.
